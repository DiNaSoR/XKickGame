# Contributing

Keep changes small and document runtime findings in `docs/RUNTIME_TROUBLESHOOTING.md`.

Before opening a change:

1. Run the relevant local check or document why it cannot run.
2. Avoid committing logs, local database dumps, generated caches, or private credentials.
3. Keep runtime artifacts in place unless the change is specifically about replacing or removing them.
4. Update `docs/DATABASE_SETUP.md` or `docs/PUBLIC_RELEASE.md` when setup or public-release policy changes.
