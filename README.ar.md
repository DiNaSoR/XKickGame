# XKickGame

مستودع قديم للعبة XKick Online يحتوي على عميل ويندوز، تشغيل السيرفر، أدوات الويب، وتجهيز قاعدة بيانات محلية.

هذا المستودع مجهز للنشر العام مع إبقاء ملفات التشغيل داخله. وجود ملفات العميل والسيرفر التنفيذية مقصود لأن السورس المسترجع غير مكتمل حاليًا، وهذه الملفات مطلوبة للتشغيل المحلي والاختبار.

## حالة المشروع

هذه النسب توضّح جاهزية التشغيل المحلي بشكل عملي. هي ليست ضمانًا بأن كل خصائص النسخة القديمة مكتملة.

الخلاصة: المستودع مكتمل بما يكفي لتشغيل العميل والسيرفر محليًا، لكنه ليس بعد سورس مفتوح قابلًا لإعادة البناء بالكامل من الصفر.

## المحتوى

- `client/XKickOnline/` - ملفات عميل ويندوز، المشغل، ملفات DLL، ملفات PAK، وسكربتات مساعدة محلية.
- `server/runtime/legacy-home/` - ملفات تشغيل سيرفر Linux القديمة، وتشمل certify/game.
- `server/runtime/docker/` - غلاف Docker لتشغيل ملفات Linux القديمة 32-bit على جهاز حديث.
- `server/runtime/mysql/` - جداول وبيانات MySQL المطلوبة للتشغيل المحلي.
- `web/` - أدوات PHP القديمة للإدارة والتحديثات.
- `data/` - جداول اللعبة المستخرجة.
- `docs/` - توثيق الإعداد والنشر وحل المشاكل.

## المتطلبات

- Windows لتشغيل عميل اللعبة.
- Docker Desktop مع Docker Compose.
- PowerShell.
- أداة `rg` / ripgrep لفحص جاهزية النشر.

## التشغيل السريع

من جذر المستودع:

```powershell
docker compose up --build
```

يستخدم Docker Compose المنافذ `3306` و `2001` و `3001`. إذا كانت هناك حاويات XKick أو MySQL قديمة تعمل على نفس المنافذ، أوقفها أولًا.

لإعادة بناء قاعدة البيانات المحلية من البداية:

```powershell
docker compose down -v
docker compose up --build
```

هذا يحذف Volume الخاص بقاعدة بيانات MySQL ثم يشغل ملفات SQL الموجودة في `server/runtime/mysql/`.

## إعداد العميل

المسار المتوقع للعميل:

```text
client/XKickOnline/
```

إذا ظهر في المشغل خطأ `Not installed`، شغل الأمر التالي من PowerShell بصلاحيات المدير:

```powershell
.\client\XKickOnline\register_xkick_install.ps1
```

بعدها شغل اللعبة من داخل `client/XKickOnline/`.

## حساب الدخول المحلي

قاعدة البيانات تنشئ حساب تجربة محلي:

```text
username: test
password: 1234
```

هذه بيانات تطوير عامة فقط. لا تستخدمها في سيرفر مكشوف على الإنترنت.

## فحص الجاهزية قبل النشر

قبل نشر المستودع أو فتح Pull Request شغل:

```powershell
.\scripts\verify-public-release.ps1
```

السكربت يفحص الملفات المطلوبة للنشر، مؤشرات القيم الخاصة القديمة، الملفات غير المناسبة للتتبع، وجود ملفات التشغيل، حدود حجم ملفات GitHub، وصحة `docker-compose.yml`.

## النشر

اقرأ `docs/PUBLISHING.md` قبل جعل المستودع عامًا.

الخطوات الموصى بها:

1. شغل `.\scripts\verify-public-release.ps1`.
2. تأكد من حقوق إعادة نشر ملفات العميل والسيرفر المرفقة.
3. استخدم تاريخ Git نظيف إذا كان التاريخ القديم قد يحتوي على كلمات مرور أو بيانات خاصة.
4. انشر إلى مستودع GitHub جديد وفارغ.
